## 前言

在1 on 1 職涯時，蠻多年輕人都會嚮往 Project manager 這角色會很感興趣，或是想把 Project manager 當成工程師的最終職涯路徑。  

但還有叫 Product manager，簡稱DM ，了解客戶需求，規劃產品發展，甚至有些公司有Program Manager，則是處理產品行銷定位，並推廣產品，但不在此篇討論的範疇。  

大多數 Project manager 管專案，不帶人，是專業職，非管理者，聽到這，有些人可能會很開心，不用管人，但要和很多專案利害關係人溝通協調，這也不是輕鬆的事，但如果要複雜的專案運作更順暢，此時Product manager角色就很重要。  

## 談談PM的幾個主要職能
### 1.需求消化過濾
收到需求後，PM需要先判斷兩件事: - 
- 為什麼使用者提出這個需求? => 找先去找需求方釐清原因 。
- 釐清需求(技術上/時程上)合理嗎? => 找RD討論 。

### 2.跨部門的溝通協調，及需求確認
- 跨部門溝通協調資源，在品質、時間、範疇找到平衡，告知需求單位自己的難處，共情原則，讓彼此都知道對方難處，團隊才能更好的support。
- 安排優先順序(Priority): 資源有限，請明確告知團隊哪幾項功能是最急的，哪些次之。
- 撰寫需求說明及目的，並跟團隊說明。

### 3.專案及資源安排及追蹤事情的進度
資淺的PM像助理，但資深PM就像導演一樣，什麼時間誰該進場，安排好相關資源，時間到演員就找劇本演戲。
- 安排好未來的專案的排程。
- 協調專案資源。
- 追蹤專案的瓶頸。

### 4. 確保成果符合使用者預期

## 結論
PM不僅需要擅長協調溝通，還需具備一定的技術背景，雖然技術背景有加分效果，但通常擁有技術背景的人不太擅長溝通最關鍵的是對這類工作感興趣並對溝通充滿熱情，否則這樣的工作可能會令人感到痛苦。
