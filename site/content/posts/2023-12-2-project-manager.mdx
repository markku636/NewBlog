## 前言
在1 on 1 職涯時，蠻多年輕人都會嚮往 Project manager 這角色會很感興趣，或是想把 Project manager 當成工程師的最終職涯路徑。  

但還有PM是叫 Product manager，又稱DM ，了解客戶需求，規劃產品發展，甚至有些公司有職稱是 Program manager，則是處理產品行銷定位，並推廣產品，但不在此篇討論的範疇。  

大多數 Project manager 管專案，不帶人，是專業職，而非管理者，聽到這，有些人可能會很開心，不用管人，但要和很多專案利害關係人溝通協調打交道，但這可不是輕鬆的事，如果要跨部門及跨組織的複雜的專案運作更順暢，此時Project manager角色就很重要。  

## 談談Project manager的幾個主要職能
### 1.需求過濾及需求確認
收到需求後，PM需要先判斷兩件事:  
- 為什麼使用者提出這個需求? => 找先去找需求方釐清原因，更有經驗的Project manager 會在討論過程中就把種種因素考量，收斂需求，讓其合理化。
- 釐清需求(技術上/時程上)合理嗎? => 找相關人員討論需求合理性

### 2.明確傳達每項需求的說明和目的
- 撰寫需求文件及確認書
- 確保所有團隊成員對目標有清晰的理解。

### 3.跨部門的溝通協調
- 管理不同部門間的資源和期望。
- 確定任務的優先順序，以最有效地利用有限的資源(在品質、時間、範疇找到平衡)。
- 安排優先順序(Priority)，正因為資源有限，請明確告知團隊哪幾項功能是最急的，哪些次之。

### 4.專案和資源的安排與進度追蹤
- 規劃和安排即將到來的專案。
- 協調專案所需的資源。
- 監控專案進度，並識別可能的瓶頸。

### 5. 確保成果符合使用者預期
最終，Project manager的目標是確保專案的結果符合利利害相關人預期和需求。

## 結論
Project manager 不僅需要擅長協調溝通，還要了解各種專業領語的知識及行話，還了解每個利害關係的在意什麼，各種角色在專案中要做些什麼，不見得需要技術背景，有技術背景是有加分效果，也會比較好帶案子，但通常擁有技術背景的人，會不太擅長溝通，最關鍵的是對這類工作感興趣，並對與人溝通打交道的工作充滿熱情，否則這樣的工作可能會令人感到痛苦。    

任何工作仍是有深度的，市場價值也不同，資淺的Project manager可能像助理，但資深Project manager就像導演一樣，什麼時間誰該進場，安排好相關資源，時間到演員就照劇本出場演戲。  

任何工作還是要有愛，沒有愛是走不長久的。
